# page-ninegames-pages

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态资源的形式存放，不针对任何特定网站或域名，仅作为页面集合的存储与分发点。

仓库名称中的 “ninegames-pages” 代表本仓库主要面向与游戏相关的页面归档，但并不限制页面内容的具体类型。

## 目录说明

```
.
├── index.html          # 仓库首页（可选）
├── pages/              # 页面归档目录
│   ├── example-page/   # 示例页面文件夹
│   │   ├── index.html
│   │   └── assets/     # 页面资源（CSS、JS、图片等）
│   └── ...
├── assets/             # 仓库级公共资源（可选）
└── README.md           # 本文件
```

- `pages/`：存放所有独立页面，每个页面一个子目录，子目录内包含该页面的 HTML 文件及其相关资源。
- `assets/`：存放可能被多个页面引用的公共资源，如通用样式、脚本或图标。
- `index.html`：仓库根目录的首页，可根据需要配置为页面导航或信息页。

## 页面归档说明

所有页面均为独立的 HTML 文档，彼此之间无数据依赖。每个页面文件夹应包含完整的自包含文件，确保在脱离仓库上下文时仍可正常浏览。

### 页面命名规范

- 文件夹名称建议使用英文小写字母、数字和连字符（`-`），避免空格和特殊字符。
- 每个页面文件夹内必须包含一个 `index.html` 文件作为入口。

### 资源引用

- 页面资源（CSS、JS、图片等）建议存放在该页面文件夹内的 `assets/` 子目录中。
- 如需引用仓库根目录下的公共资源，请使用相对路径（如 `../../assets/style.css`）。

## 维护说明

- 本仓库由维护者不定期更新，新增或修改页面时请保持目录结构一致。
- 所有页面内容由维护者自行负责，仓库不承担内容合规性审查义务。
- 如有需要，欢迎通过 Issues 提交建议或问题，但请注意本仓库不保证及时响应。
